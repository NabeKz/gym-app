# DB 設計方針

## NULL を使わない

- カラムは原則 `NOT NULL` にする
- 「値がない」状態を NULL で表現しない
- 任意の文字列フィールドは空文字 `""` をデフォルトとする
  - ただし `DEFAULT ""` は DB に持たせず、アプリ側で保証する
- 状態の有無（削除済み・確認済みなど）は別テーブルで表現する

## テーブルで状態を表現する

- ソフトデリートの `deleted_at` のような nullable カラムは使わない
- 代わりにアーカイブテーブルなど状態ごとにテーブルを分ける
- 「削除済みを含めて検索」などの横断クエリが必要なときは明示的に JOIN する

## タイムスタンプ

- カラム型は `timestamp`（タイムゾーンなし）を使う
- `timestamptz` は使わない（接続のタイムゾーン設定に依存するため）
- アプリ側で UTC に統一して保存・読み出しする責務を持つ
