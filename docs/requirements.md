# 要件定義書

スポーツジム予約管理システム

## 目的

- スポーツジムのレッスン予約をオンラインで管理できるようにする
- スタッフがレッスンスケジュールを登録し、会員がスマートフォンやブラウザから予約・キャンセルを行えることを目指す

---

## ユーザー（アクター）

| アクター | 説明 |
|---|---|
| スタッフ | レッスン・スケジュールを登録・管理する。認証はスコープ外 |
| 会員 | レッスンを検索・予約・キャンセルする。外部認証基盤でログインする |

---

## ユーザーストーリー

### スタッフ

- [x] スタッフとして、レッスンのスケジュール（日時・定員・インストラクター）を登録したい
  （週ごとにレッスン内容が変わるため）
- [ ] スタッフとして、レッスンのスケジュール（日時・定員・インストラクター）を更新・削除したい

### 会員

- [x] 会員として、開催予定のレッスン一覧を空き状況付きで確認したい
  （満席のレッスンに申し込む手間をなくしたい）
- [ ] 会員として、レッスンを予約したい
- [ ] 会員として、自分の予約を一覧で確認したい
- [ ] 会員として、不要になった予約をキャンセルしたい

---

## ビジネスルール

- 定員に達しているレッスンは予約できない
- レッスン開始の N 時間前（レッスンごとに設定）を過ぎたら予約できない
- レッスン開始の N 時間前（レッスンごとに設定）を過ぎたらキャンセルできない

---

## スコープ

### フェーズ 1（初期実装）

- レッスンスケジュールの管理（スタッフ操作）
- レッスン一覧取得・予約・キャンセル・自分の予約一覧（会員操作）

### フェーズ 2（拡張）

- キャンセル待ち登録
- キャンセル発生時の自動繰り上げ（非同期処理）

### スコープ外

- スタッフ向け管理 UI（フェーズ 1 は seed データで初期投入）
- 会員登録・プロフィール管理
- 認証基盤の構築
- 決済
