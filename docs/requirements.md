# 要件定義書 — スポーツジム予約管理システム

## 概要

スポーツジムのレッスン予約を管理するシステム。スタッフがスケジュールを登録し、会員がレッスンを予約・キャンセルできる。

---

## ユーザー

| ユーザー | 説明 |
|---|---|
| スタッフ | レッスン・スケジュールを管理する側。認証は本スコープ外 |
| 会員 | レッスンを予約する側。認証は Cognito（外部）に委譲 |

---

## エンティティ

| テーブル | 主な属性 |
|---|---|
| `lesson_masters` | レッスン名、説明、キャンセル締め切り（開始 N 時間前） |
| `instructors` | 名前 |
| `schedules` | レッスンマスタ、インストラクター、開始日時、終了日時、定員 |
| `members` | cognito_sub |
| `reservations` | スケジュール、メンバー、予約日時 |
| `cancellation_waitlists` | スケジュール、メンバー、登録日時 |

---

## 機能スコープ

### フェーズ 1（初期実装）

#### スケジュール管理（スタッフ操作）

- スケジュールの作成・更新・削除・一覧取得
- スタッフ向け管理 UI は作らず、seed データで初期投入する

#### 予約（会員向け）

- スケジュール一覧取得（空き状況付き）
- 予約作成
  - 定員超過時は予約不可
  - キャンセル締め切り後は予約不可
- 予約キャンセル
  - レッスン開始の N 時間前（レッスンマスタで設定）を過ぎたらキャンセル不可
- 自分の予約一覧取得

### フェーズ 2（拡張）

- キャンセル待ち登録
- キャンセル発生時の自動繰り上げ（非同期処理）

---

## スコープ外

- スタッフ向け管理 UI
- 会員登録・プロフィール管理
- 認証（Cognito に委譲）
- 決済

---

## API 一覧

| メソッド | パス | 説明 |
|---|---|---|
| `GET` | `/schedules` | スケジュール一覧（空き状況付き） |
| `POST` | `/schedules` | スケジュール作成 |
| `PATCH` | `/schedules/:id` | スケジュール更新 |
| `DELETE` | `/schedules/:id` | スケジュール削除 |
| `POST` | `/reservations` | 予約作成 |
| `DELETE` | `/reservations/:id` | 予約キャンセル |
| `GET` | `/reservations` | 自分の予約一覧 |
